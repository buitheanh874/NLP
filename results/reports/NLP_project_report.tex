\begin{abstract}
This report presents an NLP project for review understanding on Amazon Gift Card data. The project has three NLP layers. The first layer is sentence-level sentiment classification with context-aware text features. The second layer is multi-label issue extraction to identify concrete complaint types. The third layer is a transformer extension for deep contextual semantics and hard-case analysis. The work uses 152,410 raw reviews, with strong sentiment labels from ratings 1, 2, 4, and 5, and an uncertain analysis set from rating 3. The selected classic sentiment variant uses word and character TF-IDF with negation tagging. On test data, it reaches precision for negative class of 0.677, recall for negative class of 0.932, and F2 for negative class of 0.867. The transformer extension reaches higher overall accuracy and higher negative precision, but lower raw negative recall at fixed threshold. For issue extraction, the word-level multi-label model gets test micro-F1 of 0.765, and the word plus character model gets 0.763. To improve course alignment without adding data, the project now includes a strong syllabus upgrade package with Naive Bayes, Perceptron, LinearSVM, SGD log-loss, semantic SVD baselines, and a feed-forward MLP baseline. In that package, test recall\_0 ranges from 0.719 to 0.941, with the best F2\_0 at 0.855 for balanced Logistic Regression and the best recall\_0 at 0.941 for Complement Naive Bayes. The package also adds unigram and bigram language models with add-k smoothing, where bigram test perplexity improves from 377.45 to 362.72. A course-fit matrix built from real artifacts reports estimated syllabus coverage of 75.7\%.
\end{abstract}

\section{Introduction}
Natural language in user reviews is noisy, short, and mixed. A single review can include praise, complaint, slang, typo, and domain words in the same sentence. This makes review understanding a good NLP problem for both classic and modern methods \cite{jurafsky2026,turney2002}.

This project studies review text from Amazon Gift Cards. The goal is not only to assign a positive or negative label. The goal is to build a practical language pipeline that can understand sentiment, mark uncertain cases, and extract concrete issue tags for support work.

The project is designed as an end-to-end NLP system. It includes text normalization, lexical and subword feature modeling, context-aware handling of negation and contrast, supervised learning, uncertainty control, multi-label issue extraction, and transformer-based extension.

\section{NLP Problem Definition}
\subsection{Task A: Sentiment Understanding}
Task A is binary sentiment classification from review text. Ratings 1 and 2 are mapped to negative class. Ratings 4 and 5 are mapped to positive class. Ratings with value 3 are used for uncertainty analysis and are not part of strong supervised training \cite{turney2002,wang2012}.

The practical decision policy is not plain accuracy-first. In support triage, missing a bad review is risky. Therefore model selection follows an internal negative-first heuristic: prioritize recall for class 0, then F2 for class 0, then precision for class 0.

\subsection{Task B: Issue Extraction}
Task B is multi-label issue extraction. One review can contain more than one issue, so single-label tagging is not enough \cite{tsoumakas2011,zhang2014}. The output label set in this project has nine issue types and allows multi-label assignment per row.

This task is a core NLP mapping from unstructured review text to structured issue signals. It helps support teams move from general sentiment to action categories.

\subsection{Task C: Deep Context Extension}
Task C is transformer fine-tuning as NLP extension. The extension aims to test deep contextual language modeling for difficult short phrases and negation patterns. It also provides a direct comparison with classic sparse representations \cite{vaswani2017,devlin2019}.

\section{Text Processing Design}
\subsection{Normalization}
The core normalization pipeline uses lowercase conversion, URL removal, punctuation cleanup, and white-space normalization. This reduces token sparsity while keeping core sentiment words.

Optional abbreviation normalization maps common short forms to full forms. This includes mappings such as gr8 to great, thx to thanks, u to you, and idk to i do not know.

\subsection{Negation Handling}
Negation is a key NLP challenge in short reviews. The project uses a negation tagging method. After a negation cue, the next few tokens receive a NOT\_ prefix. The window size is three tokens by default \cite{wiegand2010}.

This method is simple but useful for sparse models. It helps separate good from NOT\_good and bad from NOT\_bad in feature space.

\subsection{Contrast Handling}
The pipeline also models simple discourse contrast. When markers such as but, however, although, though, and yet appear, text can be split into left and right clauses. Separate clause features and marker flags are then added.

This design captures a common review pattern where the second clause carries the final user opinion.

\subsection{Character-level Modeling}
Character 3 to 5 gram features are used as optional union with word features. This helps with typo, short slang, and small spelling variants. In NLP tasks on user text, this often improves robustness \cite{joulin2017}.

\subsection{Lexicon Feature Block}
The project includes an AFINN-based sentiment lexicon block in one variant. It computes raw score, negation-aware score, intensity-aware score, and contrast-weighted score. This adds explicit lexical prior to sparse representation \cite{nielsen2011}.

This part is useful for NLP course learning because it connects classic lexicon methods with modern supervised learning in one pipeline.

\section{Modeling Approach}
\subsection{Sentiment Classifiers}
The main sentiment classifier is Logistic Regression. Chi-square feature selection is applied with top K values from 1,000 to 10,000. Class weight settings include none, balanced, and weighted variants \cite{forman2003}.

Additional sentiment models include L1 Logistic Regression for sparse embedded behavior, Decision Tree, and Random Forest.

\subsection{Issue Extraction Model}
Issue extraction uses one-vs-rest training across nine labels. The default model is Logistic Regression with balanced class weight. A LinearSVM baseline is also available \cite{rifkin2004}.

Threshold tuning is done per label on validation data. This is important in multi-label NLP because one global threshold is often not enough for all labels.

\subsection{Transformer Extension}
The transformer extension fine-tunes a sequence classifier with class-weighted loss. It uses cleaned text and evaluates with the same uncertainty and threshold framework for direct comparison \cite{devlin2019}.

This extension gives deeper contextual semantics and helps analyze hard short phrases where sparse models may fail.

\section{Experimental Setup}
\subsection{Split and Protocol}
All experiments use seed 42. Strong sentiment labels are split with train, validation, and test ratio 70, 10, and 20. Three-star reviews are held for uncertainty analysis.

Issue extraction uses 19,000 labeled rows with split sizes 13,300 for train, 1,900 for validation, and 3,800 for test in saved runs.

\subsection{Main Metrics}
For sentiment, key metrics are precision\_0, recall\_0, F1\_0, and F2\_0, along with full accuracy and F1. Because the strong-label set is imbalanced, model trade-offs are interpreted mainly through precision-recall behavior rather than ROC-only views \cite{saito2015}. Selective metrics under uncertainty include coverage, uncertain rate, selective precision\_0, selective recall\_0, selective F2\_0, and false-negative rate on covered samples.

For issue extraction, key metrics are micro-F1, macro-F1, subset accuracy, hamming loss, and per-label precision, recall, and F1.

\subsection{Uncertainty Rule}
The uncertainty policy uses low threshold and high threshold on positive probability. It follows a selective classification abstention setup and can return uncertain in the middle range \cite{geifman2017}. We apply an additional heuristic in the classic pipeline that returns uncertain for too short or sparse text cases.

This rule makes NLP output safer for production because low-information text is not forced into a hard wrong class.

\subsection{Cross-understanding, Review, and Reproducibility}
The workflow is designed so that each critical module can be executed by at least one member who is not the original author. For each module handover, the receiving member reruns the pipeline from saved commands and checks three gates: no train-test leakage, fixed random seed behavior, and consistency between table values and exported CSV files.

For code quality control, we use a branch-and-review policy with one non-author review per substantial change. The review checklist focuses on metric correctness, split integrity, and reproducibility. This process aligns with course criteria on module split, shared understanding, and cross-review practice.

\subsection{Defense Evidence Bundle}
To make teamwork evidence auditable in the defense session, the bundle includes: role/module matrix, command logs for reruns, model-output CSV artifacts, and review notes linked to changed modules. This evidence complements the quantitative model results and helps justify process quality, not only final scores.

\section{No-New-Data Syllabus Upgrade Package}
The project now includes an extra package that improves alignment with NLP course topics without adding any new labeled data. The package reuses the same train, validation, and test split, and keeps the same text cleaning policy.

The first upgrade block is a strong classic benchmark suite. It includes balanced Logistic Regression, Multinomial Naive Bayes, Complement Naive Bayes, Perceptron, LinearSVM, SGD with log-loss, semantic Logistic Regression on SVD vectors, and a feed-forward MLP on the same SVD vectors. The train size is capped at 35,000 rows for this block to keep runtime practical while preserving class ratio.

The second upgrade block is a language model block. It trains unigram and bigram language models with add-k smoothing and reports perplexity on validation and test splits. This gives direct evidence for the N-gram language model topic in the course \cite{jurafsky2026,manning2008ir}.

The third upgrade block builds a topic-coverage matrix from project artifacts. The matrix uses explicit topic scores and reports an overall syllabus coverage score for defense discussion.

\section{Classic Versus Transformer Discussion}
Classic sparse NLP and transformer NLP provide different strengths. Classic V6 keeps high negative recall and strong selective recall in this project setting. Transformer gives higher global accuracy and stronger negative precision.

For support triage that needs to catch as many negatives as possible, classic V6 is more aligned with the target policy. For semantic nuance and idiom handling, transformer can be useful and can be combined with rule-based control.

A practical hybrid path is possible. One option is to keep classic model as primary triage model and use transformer as secondary scorer on hard uncertain cases.

\section{Demo Design for Defense}
The live demo is designed for a 10-minute defense with stable and auditable behavior. The main flow is: input review text, run classic sentiment with uncertainty, optionally run transformer comparison, then trigger issue extraction when review is negative or needs attention.

For UI delivery, the project provides a lightweight interactive app and keeps CLI fallback scripts for reliability. The demo script uses three core scenarios: a clear positive review, a clear negative review with issue tags, and an ambiguous short review that should return uncertain with reason. This scenario-based demo helps examiners verify not only accuracy but also safety behavior and practical usability.

\section{Practical Value for NLP Course}
This project now shows an even wider set of NLP ideas in one system. It covers text normalization, lexical mapping, n-gram sparse encoding, negation scope handling, clause-level cues, multi-label text classification, threshold calibration, and transformer fine-tuning. It now also includes explicit Naive Bayes, Perceptron, and N-gram language model artifacts, plus a feed-forward network baseline on semantic SVD vectors.

The project remains practical for course work. It has saved artifacts, reproducible metrics, an annotation workflow, and deployable inference scripts. It also has a topic-coverage matrix that can be shown directly in defense.

For course learning, the updated version provides stronger evidence of trade-offs between classic and deep NLP approaches under one business objective and one fixed dataset.

\section{Limitations and Future Work}
The main limitation is still data imbalance, especially in negative sentiment and rare issue labels. This affects precision and stability in several models.

Another limitation is topic coverage depth. The project now covers more course topics, but RNN, masked language model tasks, and LLM applications are still light compared with the rest.

Future work can still avoid new data at first. It can add a small RNN baseline on the same split, add a masked language model probe with the same text corpus, and add a simple retrieval or prompt-based LLM application demo. This will improve topic coverage while keeping the same dataset.

\section{Conclusion}
This NLP project builds a complete review understanding pipeline from sentiment classification to issue extraction and transformer extension. Classic context-aware sparse modeling gives strong negative-first performance. Transformer extension gives better global precision and useful semantic behavior in some hard phrases. Multi-label issue extraction provides detailed action signals from review text. The upgraded syllabus block adds Naive Bayes, Perceptron, feed-forward semantic baselines, and N-gram language model evaluation without adding data. With this upgrade, the project gives stronger alignment with course topics and remains reproducible for defense and practical use.
